# May 6: SpaceX Signs, Rate Limits Double

Anthropic announced doubling of Claude Code rate limits tied to a deal with SpaceX. After the session ceiling issues from last week, this is very needed.

<!-- more -->
<!-- enriched -->

## Developer Journal

### Rate limits doubling

Anthropic is doubling Claude Code rate limits. The deal with SpaceX apparently drove the capacity expansion — which is an interesting signal about where the enterprise demand is coming from — but the benefit extends to all Claude Code subscribers, not just aerospace customers.

The direct relevance: April 30's mid-morning session limit and the ongoing frustration of hitting ceilings at inconvenient moments. A 2x increase in limits means either twice as many sessions before hitting the wall, or longer/heavier individual sessions that currently get cut short.

For the JWST project's workflow specifically, the limiting factor isn't raw token count — it's context size and session depth. The `/goal` mode runs and multi-file refactor sessions are the ones that run into limits, and those are also the sessions where the limit interruption is most costly (rebuilding context in a new session takes time). More headroom there is directly useful.

### Worth watching: enterprise AI infrastructure

The SpaceX connection is worth noting beyond the rate limit announcement. SpaceX is one of the more demanding engineering environments on the planet in terms of code complexity, real-time constraints, and safety requirements. An Anthropic/SpaceX partnership implies Claude Code is being used for serious production engineering, not just prototyping.

That's relevant for evaluating how much to lean into Claude for safety-critical adjacent code paths in the JWST project — the auth layer, the MAST data validation, the composite memory management. If the tool holds up at SpaceX scale, the confidence bar for complex use cases goes up.
