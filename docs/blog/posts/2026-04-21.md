# April 21: ChillPill, Auto Mode, and the Economics of Fan Control

A side project for MacBook thermal management, Auto Mode goes live in Claude Code, and the token count conversation gets interesting.

<!-- more -->
<!-- enriched -->

## Developer Journal

### ChillPill: the $15 problem and the $100 solution

The MacBook overheats. Mac Fan Control only offers two speeds — normal or full blast — and unlocking arbitrary RPM control costs $15 for the pro version. That's the kind of narrow, well-defined scratch-your-own-itch problem that Claude handles well.

A couple of turns in Claude Code produced a working POC: a small macOS taskbar app that exposes fan RPM controls directly. The irony wasn't lost on anyone — Mac Fan Control wants $15 to unlock RPM control, and Claude wants $100/month to build the workaround. The math only pencils out if you're using Claude for substantially more than one fan controller. In this case: yes.

The ChillPill experiment also felt like a good stress test for autonomous development on a low-stakes project. No production users, no schema migrations, no concurrency hazards — just build the thing until it works. The tight feedback loop (run, observe fan behavior, tweak) made it a satisfying one-session project.

### Auto Mode arrives

Auto Mode became available in Claude Code today — and in the desktop app as well. The previous workaround, "bypass permissions mode," is effectively a whitelist of everything; Auto Mode introduces actual safety reasoning into the loop while still letting long-running sessions proceed without constant approvals.

The trade-off is real: Auto Mode is safer but still room for issues compared to a fully supervised session. For side-project work and well-scoped tasks it hits a useful middle ground. The first production use cases that feel right: dependency updates, code cleanup sweeps, and anything where the blast radius is bounded by a test suite.

The two-billion-token-in-a-week discussion that came up today is worth noting: that's a lot of compute, and having better autonomous tooling is increasingly relevant as the JWST project scales. The session limits hit harder when each session is doing more.
