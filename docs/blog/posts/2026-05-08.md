## Developer Journal

### Composite async streaming: the full picture

This was the week the async composite wizard came fully together. The pieces shipped in order:

**Async preview** — the composite preview step now starts a background job rather than blocking the request thread. The wizard UI transitions to a "generating..." state while the job runs. Previously, generating a composite preview was a synchronous request that could take 30+ seconds for large recipes — timeouts, connection drops, and user confusion about whether something had failed.

**Real-time engine streaming with live log panel** — the processing engine now streams job status events back to the backend via a persistent connection. The backend relays these to the frontend via SignalR. The result: the composite wizard shows a live log panel during generation — filter loads, normalization steps, memory usage, completion percentage. Users can watch the job progress in real time instead of looking at a spinner and hoping.

The `JobStatus.cs` model touched three times during the week as the event schema evolved — adding fields for progress percentage, current step label, and the warnings that will propagate to the header.

**Warning header propagation** — composite generation can succeed but still have warnings (e.g., a filter was silently downscaled, a channel was skipped due to data quality issues). Previously these warnings lived only in the server logs. Now they surface as response headers, the frontend reads them, and the composite result shows them inline. This closes the loop on the "silent stretch fallback" category of issues.

### Recipe spatial-grouping bug (NGC-346)

A bug in curated recipe spatial grouping caused the NGC-346 patchwork to produce visually incorrect results — some exposures were being grouped into the wrong spatial tile. The fix was in the processing engine's grouping logic: the tile assignment was using bounding box overlap rather than centroid proximity, which works for non-overlapping exposures but breaks when MAST returns tiles with significant overlap margins. Switched to centroid-based assignment with an overlap tie-breaking rule.

### `CompositePreviewStep.tsx` hotspot

That component touched 3 times this week as the async epic landed. The three passes: initial async job kick-off, adding the live log panel, wiring the warning header display. Each pass made the component heavier, so the next time it's touched it probably deserves a look at extraction (the log panel is a clear candidate for its own component).
