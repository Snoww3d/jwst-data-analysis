## Developer Journal

### The memory-budget epic: why it took a saga

NGC-3324 is one of the larger targets in the dataset. Generating its composite was failing with out-of-memory errors under realistic server constraints — the in-memory representation of 158+ FITS files at full resolution is enormous. The fix couldn't just be "allocate more memory"; it needed to be a real solution that worked within the resource bounds of the production environment.

The solution ended up spanning every layer of the stack:

**Processing engine** — hybrid downscale approach: FITS files above a resolution threshold get downscaled during ingestion rather than at composite-generation time. The threshold is tuned so the quality loss is negligible at standard viewing sizes but the memory footprint drops significantly. The "hybrid" part means small files still get processed at native resolution, so detail isn't sacrificed uniformly.

**Backend API** — new `/api/composite/estimate` endpoint. Takes the recipe parameters and returns the estimated memory budget (and a flag for whether it exceeds the server's working limit) without actually running the composite. This lets the frontend warn the user before they start a job that's going to fail.

**413 propagation** — when a composite job does exceed budget (whether the user was warned or not), the backend now returns a proper 413 with structured error details rather than a generic 500. The frontend parses the error type and shows a specific "resource limit exceeded" message instead of a confusing generic failure.

**Warning banner** — before the composite starts, the wizard checks the estimate. If it's over-budget, a warning banner appears with options. The user can reduce the recipe (remove filters, lower resolution) or...

**"Continue anyway" override** — ...proceed knowing it might fail or be slow. Power users who know their server has headroom can bypass the budget check. This was a deliberate choice: enforcing the limit hard would make the app feel restrictive, but ignoring the limit leads to silent OOM crashes. The warning + override threads the needle.

### Wavelength ribbon bonus

A smaller but tidy addition: the Process step now shows a wavelength ribbon — a visual bar spanning the filters selected for the composite, colored by wavelength. At a glance you can see whether you're working in the infrared end of the spectrum or spanning NIRCam to MIRI. It's the kind of context that the wizard was missing — JWST's filter names (`F150W`, `F444W`) are useful but not immediately intuitive without a reference.

14 of the 19 PRs were dependency maintenance. That's a high ratio, but this week it felt fine because the 5 feature/fix PRs were high-signal. The dependency queue had accumulated from the previous week's security sprint. Clearing it keeps the Dependabot noise manageable going into next week.
