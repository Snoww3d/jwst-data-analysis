## Developer Journal

### JWST Discovery Design System lands (+3,524 lines)

The design system foundation is in. This has been the missing piece for consistency across the wizard flows — every new component since January has been hand-rolling spacing, color, and typography. The system brings UI primitives into a single source of truth and starts retiring one-offs like AuthToast in favor of the design token approach.

3,524 lines is a big drop for a foundation layer, but the math is mostly additive (tokens, primitives, migration stubs) rather than net-new UI logic. The important thing is that the shape is now defined. Older components can migrate incrementally without blocking new work.

### Security hardening: sanitizer and enumeration patches

Two security items closed this week:

**Filename sanitizer vs URL-encoded traversal** — the existing sanitizer was checking for literal `../` sequences but not URL-encoded variants like `%2e%2e%2f`. Added normalization before the check so both forms are caught. Discovered via a review of OWASP path traversal test vectors, not an incident.

**Mosaic generation enumeration** — `GenerateAndSaveMosaic` was leaking information about whether a target existed or not through its error codes (404 vs 403 behaving differently). Normalized the response so "not found" and "not authorized" both return 404. Auth-denial logging was also restored to Warning level — it had been silently downgraded to Debug in a previous refactor, making unauthorized access attempts invisible to operators.

These patch two of the three security epics that were specced out at the end of March. The third (apiClient auth hardening) is still in the backlog as a larger planned PR.

### Production Docker environment config

The production environment was failing to boot because `appsettings.Production.json` still pointed to `localhost` addresses — correct for local dev, wrong for Docker Compose where services resolve by container name. Set `backend`, `processing-engine`, and `mongo` as defaults matching the `docker-compose.yml` service names. Env vars still override when set. Also shipped backup/restore scripts and a documented deploy workflow alongside this.
